# 并行节点

## 节点说明

并行节点用于同时触发多个任务并发执行，提升流程的处理效率，适用于多个独立任务可同时处理的场景。

## 使用方式

* 点击"添加并行任务"按钮，可以添加多个子任务。
* 每个子任务可选择一个工具作为执行单元。
* 每个工具都可以独立配置参数，互不影响。
* 所有并行任务将被同时触发执行，互不依赖，彼此独立完成。

## 输出说明

* 并行节点输出为一个名为 `output` 的数组对象。
* 每个 key 为工具的名字，value 为对应工具执行后的结果。
* 若存在两个同名工具，第二个工具名称后会自动加后缀 `_1`，依此类推（例如 tool, tool_1, tool_2）。

## 错误处理

* 若某个工具在执行过程中报错，则并行节点整体会立即中止执行，标记为失败。
* 并行节点将返回第一个发生错误的工具的报错信息，供用户排查。

## 适用场景

* 同时向多个系统发起请求，例如并发调用多个API接口。
* 并行处理多份数据，例如对多个文件同时提取信息。
* 并行执行多个业务模块，例如AI分析、标签识别、数据汇总等任务。
