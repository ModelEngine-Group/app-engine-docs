# 变量聚合节点

## 节点说明

变量聚合节点用于整合多个分支中输出的变量，确保流程无论在哪个分支执行，最终都能通过统一的变量进行访问。适用于条件判断或并行执行后需要统一变量名进行后续处理的场景。

## 节点功能

* 支持整合来自不同分支的同类型变量。
* 聚合变量的类型必须一致，例如都是字符串、都是数组或对象等。
* 在单分支场景下，以最后一个聚合变量的值为准。

## 模块结构

### 1. 输入模块

* 配置多个变量值作为输入。
* 所有输入变量类型需一致，否则会提示配置错误。
* 每个变量来源可通过引用方式设置，如引用条件分支节点输出的结果。

### 2. 输出模块

* 聚合后的统一变量名固定为 `output`。
* `output` 的类型与所聚合变量的类型保持一致。

---

变量聚合节点让流程结构更清晰，避免因条件分支导致的变量命名不统一问题，是复杂流程管理中不可或缺的节点之一。
