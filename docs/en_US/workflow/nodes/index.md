# Node Introduction

Workflow nodes are the fundamental building blocks that enable intelligent automation. Each node performs a specific function and can be connected to create complex processing pipelines.

## Node Categories

### Input Nodes
- **Start Node**: Initiates workflow execution
- **Question Classification Node**: Routes user queries to appropriate paths

### Processing Nodes
- **LLM Node**: Integrates AI model capabilities
- **Code Node**: Executes custom code logic
- **HTTP Call Node**: Makes external API calls
- **Knowledge Retrieval Node**: Accesses knowledge base information

### Control Nodes
- **Condition Node**: Implements conditional logic
- **Loop Node**: Handles repetitive tasks
- **Parallel Node**: Executes multiple tasks simultaneously

### Output Nodes
- **End Node**: Delivers final results
- **Intelligent Form Node**: Presents structured output forms

## Node Configuration

Each node can be configured with specific parameters:
- Input mappings and data transformations
- Model selection and prompt engineering
- Error handling and fallback mechanisms
- Performance optimization settings
